## 2. Архитектура модуля

### Основная сущность — TaskReport
- **Таблица**: `ttt_backend.task_report`
- **Состояния**: REPORTED → APPROVED / REJECTED
- **Поля**: task, executor, reporter, approver, state, reportDate, actualEfforts, comment

### Ключевые бизнес-правила
1. **Макс. часов в день**: 36 (валидация на фронте и бэке)
2. **Месячная норма**: ~166 ч (8ч × 20.75 дн)
3. **Формат задачи**: "Проект / Задача"
4. **Период**: OPEN → можно редактировать, CLOSED → только просмотр
5. **Auto-reject**: при закрытии периода не-APPROVED → REJECTED
6. **Permissions**: REPORTS_VIEW, REPORTS_EDIT, REPORTS_APPROVE

### Ключевые endpoints
| Endpoint | Метод | Описание |
|----------|-------|----------|
| /v1/reports | GET | Поиск отчётов с фильтрами |
| /v1/reports | POST | Создание отчёта |
| /v1/reports | PUT | Batch создание/замена |
| /v1/reports/{id} | PATCH | Редактирование (состояние, комментарий, часы) |
| /v1/reports | DELETE | Удаление отчётов |
| /v1/reports/summary | GET | Месячная сводка (норма, отработано, дефицит) |
| /v1/reports/total | GET | Итого за неделю |
| /v1/reports/accounting | GET | Данные для бухгалтерии |

### Фронтенд-компоненты
- ReportPage → TaskAddContainer + WeekSwitcher + TaskReportsTable + RejectedReports
- Redux ducks: dataReducer, filtersReducer, uiFiltersReducer
- Цветовая кодировка: белый=REPORTED, зелёный=APPROVED, красный=REJECTED, серый=CLOSED
