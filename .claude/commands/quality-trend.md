Покажи тренд качества тестов $ARGUMENTS:

## 1. Загрузи историю запусков

Прочитай файл `reports/history.json`. Если файл не существует или пуст:
- Сообщи что история пуста
- Предложи запустить тесты: `npx playwright test --project=e2e --workers=2`
- После запуска history-reporter автоматически сохранит результаты

## 2. Анализ по модулю (если $ARGUMENTS указан)

Если указан модуль (например `admin`, `report`, `vacation`):
- Отфильтруй записи по модулю
- Покажи тренд pass rate за последние N запусков
- Покажи самые нестабильные тесты (flaky — проходят/падают в разных запусках)

## 3. Общий тренд (если $ARGUMENTS пуст или `all`)

### 3.1 Таблица трендов

```markdown
| Дата | Тестов | Passed | Failed | Pass Rate | Δ Rate | Длительность |
|------|--------|--------|--------|-----------|--------|--------------|
```

- `Δ Rate` — изменение pass rate по сравнению с предыдущим запуском (↑ или ↓)
- Покажи последние 10 запусков (или все если меньше 10)

### 3.2 Тренд по модулям

```markdown
| Модуль | Последний | Предыдущий | Тренд | Нестабильных |
|--------|-----------|------------|-------|--------------|
```

Для каждого модуля:
- Сравни pass rate последнего запуска с предыдущим
- Подсчитай flaky тесты: тесты которые меняют статус между запусками

### 3.3 Топ-10 нестабильных тестов

Найди тесты которые:
1. Падали чаще всего за последние 5 запусков
2. Меняли статус (passed↔failed) между запусками (flaky)

```markdown
| # | Тест | Модуль | Падений/Запусков | Flaky | Последняя ошибка |
|---|------|--------|-----------------|-------|------------------|
```

### 3.4 Рекомендации

На основе трендов предложи:

1. **Критические регрессии** — модули где pass rate снизился на >5%
2. **Стабилизация** — flaky тесты для починки (приоритет P0)
3. **Улучшение** — модули с самым низким pass rate для фокуса
4. **Оценка прогресса** — как менялось общее покрытие за последние запуски

## 4. Сравнение с QA-отчётами

Если существуют файлы `_qa-report-*.md`:
- Сопоставь модули из history.json с QA-рейтингами (A/B/C/D)
- Покажи улучшились ли модули с рейтингом D после добавления тестов
- Предложи запустить `/analyze-module [module]` для обновления рейтинга
