## Список модулей (порядок выполнения)

1. auth

## Если $ARGUMENTS содержит число N

Запусти только первые N модулей из списка (для тестирования).

## Для каждого модуля

1. Выполни полный `/analyze-module [module]`
2. Дождись завершения и сохранения `_qa-report-[module].md`
3. Запиши краткий статус: модуль, количество тестов, количество проблем, рейтинг качества
4. Перейди к следующему модулю

## По завершении всех модулей

