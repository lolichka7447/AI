## 2. Архитектура аутентификации

### Обзор
TTT использует **гибридную систему аутентификации**:

1. **Frontend (порт 9584)**: Session-based CAS (Central Authentication Service)
   - CAS Server: `https://cas-demo.noveogroup.com`
   - Feature-toggle `cas-{profile}` управляет включением CAS
   - На тестовых окружениях (qa1, qa2, stage) CAS можно отключить → форма логина

2. **Backend API (порт 9583)**: Stateless JWT + API Tokens
   - JWT: RS256 (RSA), header `TTT_JWT_TOKEN`, срок жизни — 1 день
   - API Token: header `API_SECRET_TOKEN` или query param `?token=`
   - SessionCreationPolicy: STATELESS

3. **Dev/QA endpoint**: `GET /authorization/jwt/new?login={login}` — генерация JWT без пароля (только не-production профили)

## 4. Перекрёстный анализ — найденные несоответствия

| # | Серьёзность | Описание | Рекомендация |
|---|-------------|----------|-------------|
| 1 | **Critical** | Logout НЕ тестируется — TC-NAV-011 только проверяет видимость кнопки, не кликает | Добавить deep-тест: клик Logout → редирект на CAS → нет доступа к защищённым страницам |
| 2 | **Critical** | 0 тестов на ролевой доступ — все тесты под pvaynmaster (все роли) | Добавить тесты: employee без доступа к admin, contractor с ограничениями |
| 3 | **Critical** | 0 тестов на JWT expiration/invalidation | Добавить: просроченный токен → 401 → редирект на логин |
| 4 | **Major** | 0 тестов на API Token аутентификацию | Добавить: создание токена → использование → отзыв → проверка 401 |
| 5 | **Major** | TC-NAV-011 содержит `expect(typeof isVisible).toBe('boolean')` — бесполезная проверка | Переписать: `expect(logoutItem).toBeVisible()` + реальный клик |
| 6 | **Major** | 0 тестов на disabled employee | Добавить: отключённый сотрудник → вход невозможен |
| 7 | **Major** | 0 тестов на read-only mode | Добавить: readOnly=true → запрет модификации данных |
| 8 | **Major** | 0 тестов на session timeout | Добавить: устаревшая сессия → редирект на логин |
| 9 | **Medium** | user-settings: 84% shallow тестов — только visibility | Переписать: реальные workflow (генерация токена, копирование, отзыв) |
| 10 | **Medium** | 0 тестов на 403 error page | Добавить: доступ к запрещённому ресурсу → NotAuthorizedContainer |
| 11 | **Medium** | 0 тестов на concurrent sessions | Добавить: 2 сессии одного юзера → обе работают (unlimited sessions) |
| 12 | **Minor** | CAS single sign-out не тестируется | Добавить: logout в одном табе → другой таб тоже разлогинен |
| 13 | **Minor** | CORS полностью открыт (all origins) | Security concern: проверить что CORS ограничен в production |
