Проверь покрытие тестами для модуля $ARGUMENTS:

## 1. Собери тест-кейсы

**Источники (в порядке приоритета):**
- Файл `_qa-report-$ARGUMENTS.md` (если существует)
- Confluence MCP `atlassian` → найди тест-кейсы для модуля
- Если ничего нет — предложи сначала запустить `/analyze-module $ARGUMENTS`

## 2. Найди существующие автотесты

Ищи в директориях:
- `tests/e2e/` — UI-тесты (Playwright TypeScript)
- `tests/api/` — API-тесты
- `tests/db/` — DB-тесты
- `_target-project/autotests/` — Python API-тесты (pytest)
- `_target-project/**/src/test/` — unit/integration тесты в Java/Kotlin

**Для Python API-тестов (`_target-project/autotests/`):**
- Найди файлы `test_*.py` и `*_test.py` относящиеся к модулю $ARGUMENTS
- Определи покрытые эндпоинты из assert-ов и request-ов
- Проверь тестовые данные в `_target-project/autotests/data/`
- Определи качество: реальные assertions vs `assert True`

Для каждого найденного теста определи:
- Какой тест-кейс он покрывает (по ID или описанию)
- Тип теста (unit, integration, e2e, api)
- Статус: проходит / падает / отключён
- Качество: реальная проверка / заглушка

## 3. Получи тикеты из GitLab

**Если подключён MCP `gitlab`:**
- Получи issues из `noveo-internal-tools/ttt-spring` для модуля $ARGUMENTS
- Фильтруй по labels: `$ARGUMENTS`, `bug`, `feature`, `enhancement`
- Определи requirements из открытых issues
- Сравни milestones с текущим покрытием

**Для каждого тикета определи:**
- Есть ли автотест (E2E или API) покрывающий эту функциональность
- Если баг — есть ли регрессионный тест
- Если feature — покрыта ли функциональность

## 5. Выведи приоритизированный отчёт

1. **Общая статистика**: X из Y тест-кейсов автоматизированы (Z%)
   - E2E покрытие: A%
   - API покрытие: B%
   - Оба уровня: C%
2. **Непокрытые P0**: КРИТИЧНО — список тест-кейсов без автотестов
3. **Непокрытые P1**: ВАЖНО — список
4. **Только API (нет E2E)**: список фич покрытых API-тестами, но без UI-тестов
5. **Тикеты без тестов**: GitLab issues без привязанных автотестов
6. **Непокрытые P2/P3**: ЖЕЛАТЕЛЬНО — список
7. **Рекомендация**: предложи запустить `/generate-tests $ARGUMENTS` для непокрытых P0/P1
